# Compound Geometries

Compound geometries create complete, closed profiles in a single call. They are ready to extrude right away.

## rect

```js
sketch("xy", () => {
    rect(100, 60)              // 100 wide, 60 tall, starting at current position
    rect(80)                   // 80x80 square
    rect([10, 20], 50, 30)    // 50x30 rectangle starting at position [10, 20]
})
```

Useful methods on `rect`:

<CodeBlock language="js">{compoundRect}</CodeBlock>

![Rounded centered rectangle](/img/docs/sketching/compound-rect.png)

You can also access individual edges of a rect:

```js
const r = rect(100, 60)
r.topEdge()        // the top edge
r.bottomEdge()     // the bottom edge
r.leftEdge()       // the left edge
r.rightEdge()      // the right edge
```

## slot

An obround shape (rectangle with rounded ends):

<CodeBlock language="js">{compoundSlot}</CodeBlock>

![Slot shape](/img/docs/sketching/compound-slot.png)

Slots always build along the sketch's local X axis. To orient them differently, pass a rotation angle in degrees:

```js
slot(100, 20).rotate(45);   // rotate the slot's primary axis by 45°
```

### Slot from edge

You can create a slot that follows an existing edge or arc. Pass the geometry as the first argument and the width as the second:

<CodeBlock language="js">{compoundSlotEdge}</CodeBlock>

![Slot from edge](/img/docs/sketching/compound-slot-edge.png)

The third argument (`true`) removes the source geometry after creating the slot. Set it to `false` to keep the original edge.

## text

`text()` turns a string into outline geometry — each letter becomes a closed profile you can extrude into 3D lettering. Use it inside a `sketch()` with `text("Hello")`, or standalone on a plane with `text("xy", "Hello")`. The size is the cap height in model units; thickness comes from `extrude()`.

<CodeBlock language="js">{compoundText}</CodeBlock>

![Text outline](/img/docs/sketching/compound-text.png)

### Fonts and weight

By default `text()` uses a system font. Pass a **system font name** to `.font()`, or a **workspace-relative font file** ending in `.ttf`/`.otf` (e.g. `.font("fonts/Brand.ttf")`). Control the typeface with `.weight()` (`"regular"`, `"bold"`, or a number 100–900) and `.italic()`:

<CodeBlock language="js">{compoundTextFonts}</CodeBlock>

![Bold serif text from a named system font](/img/docs/sketching/compound-text-fonts.png)

If the named font isn't installed, FluidCAD substitutes the closest match (e.g. `"Arial"` → a system sans-serif) and ultimately falls back to a default font.

### Extruding text

Because `text()` is ordinary sketch geometry, you extrude it like any other profile — the counters (the holes in `o`, `e`, `A`, …) are kept automatically:

<CodeBlock language="js">{compoundTextExtruded}</CodeBlock>

![Extruded 3D text](/img/docs/sketching/compound-text-extruded.png)

For multi-line strings (`\n`), set `.align("left" | "center" | "right")`, and fine-tune spacing with `.lineSpacing()` and `.letterSpacing()`.

### Text along a path

`text(string, path)` lays the glyphs along a curve instead of a straight baseline. The path can be a curve in the same sketch, a whole sketch, a standalone primitive (e.g. `circle("xy", 80)`), or edges selected from existing geometry (e.g. `select(edge().circle())` for a cylinder rim). The path must be planar; the text lies in the path's plane.

<CodeBlock language="js">{compoundTextPath}</CodeBlock>

![Text along an arc](/img/docs/sketching/compound-text-path.png)

Like other consuming features, the text consumes its path: the curve becomes the baseline and is removed from the scene, so it never leaks into an extruded profile. Mark the path `.reusable()` to keep it — e.g. to lay several texts on one path, as the flip and offset examples below do.

Glyph spacing is measured as arc length, so kerning and `.letterSpacing()` behave exactly like straight text. To slide the text start along the path by an arc-length distance, use `.startAt(d)`.

#### Aligning on a path

`.align()` positions the run against the path: `"start"` begins at the path's start, `"center"` centers it on the midpoint, `"end"` finishes at the path's end, and `"stretch"` justifies the glyphs evenly across the whole path (`"left"`/`"right"` work as synonyms of `"start"`/`"end"`):

<CodeBlock language="js">{compoundTextPathAlign}</CodeBlock>

![Path text alignment: start, center, end, stretch](/img/docs/sketching/compound-text-path-align.png)

#### Outside or inside: flip

On a closed path (a full circle) the text wraps around and sits on the **outside** by default — `"stretch"` then spaces the glyphs evenly around the loop. `.flip()` mirrors the text to the other side of the path and reverses the reading direction: on a circle it moves the text inside; on an open curve it mirrors the text below the path:

<CodeBlock language="js">{compoundTextPathFlip}</CodeBlock>

![Text outside and inside a circle](/img/docs/sketching/compound-text-path-flip.png)

#### Floating off the path: offset

`.offset(d)` shifts the baseline perpendicular to the path — positive values float the text above the curve, negative values tuck it below:

<CodeBlock language="js">{compoundTextPathOffset}</CodeBlock>

![Text offset above and below an arc](/img/docs/sketching/compound-text-path-offset.png)
