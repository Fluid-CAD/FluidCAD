# Transforms

Transforms move, rotate, and mirror existing geometry in 3D space. They come in two flavors:

- **Free-function form** — `translate(...)`, `rotate(...)`, `mirror(...)`. Adds a distinct entry to scene history and can copy with the `true` flag. Covered on this page.
- **Chained form** — `foo().translate(...).rotate(...)` on any builder's return value. Places a shape in one expression without adding a separate history entry. See the [Chained Transforms guide](./chained-transforms.mdx).

## Translate

Move a solid to a new position:

```js
import { cylinder, sphere, translate } from 'fluidcad/core';

cylinder(50, 100)
sphere(50)

translate([150, 0, 0], true, cylinder)
```

Signatures:

```js
translate(x, y, z)                 // move by x, y, z
translate([x, y, z])               // same, with an array
translate([x, y, z], true)         // copy instead of move (keeps the original)
translate([x, y, z], true, target) // copy a specific object
```

The third argument `true` creates a **copy** — the original stays in place and a duplicate appears at the new position.

## Rotate

Rotate geometry around an axis:

<CodeBlock language="js">{transformRotate}</CodeBlock>

![Rotated copies](/img/docs/guides/transform-rotate.png)

### Defining axes

```js
import { axis } from 'fluidcad/core';

axis("x")                                 // X axis at origin
axis("z")                                 // Z axis at origin
axis("z", { offsetX: 50, offsetY: 30 })   // Z axis offset from origin
```

## Mirror

Mirror geometry across a plane:

<CodeBlock language="js">{transformMirror}</CodeBlock>

![Mirrored shape](/img/docs/guides/transform-mirror.png)

You can use any plane name (`"xy"`, `"xz"`, `"yz"`, `"front"`, `"top"`, `"right"`, etc.) or a custom `plane()` object.

:::info[Inside a sketch?]
`mirror()` and `rotate()` also have 2D variants that transform sketch geometry directly, and `copy()` works inside a sketch too. See the [2D Transforms guide](./sketching/transforms).
:::
